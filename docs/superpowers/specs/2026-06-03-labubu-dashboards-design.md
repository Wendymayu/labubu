# Labubu Metrics Dashboard Design

> 2026-06-03 | Status: Design Approved

**Goal:** Add a Grafana-like metrics dashboard to Labubu, allowing users to create panel configurations via UI, which drive PromQL queries against the backend and render charts.

## Architecture

```
┌─────────────────────────────────────────────────────┐
│  Frontend (Vue 3 + Chart.js)                        │
│                                                     │
│  /dashboards  ─ Dashboard.vue                       │
│    ├─ TimeRangePicker (global, shared by all panels) │
│    ├─ "New Panel" button  →  PanelForm.vue (modal)  │
│    └─ CSS Grid (2 cols) of PanelChart.vue           │
│         ├─ line  → Chart.js line chart              │
│         ├─ bar   → Chart.js bar chart               │
│         └─ stat  → big number + label               │
└──────────────┬──────────────────────────────────────┘
               │ HTTP REST
┌──────────────▼──────────────────────────────────────┐
│  Backend (Go)                                       │
│                                                     │
│  /api/v1/dashboards       CRUD panel configs        │
│  /api/v1/dashboards/{id}  (JSON files in             │
│                              data/dashboards/)       │
│  /api/v1/metric-names     list known metric names   │
│                                                     │
│  /api/v1/query             instant query (stat)      │
│  /api/v1/query_range       range query (line/bar)   │
│  /api/v1/labels            label discovery          │
│  /api/v1/label/:name/values                          │
└──────────────┬──────────────────────────────────────┘
               │
┌──────────────▼──────────────────────────────────────┐
│  data/dashboards/*.json    (one file per panel)      │
└─────────────────────────────────────────────────────┘
```

**Tech Stack:** Go `net/http`, Vue 3 Composition API, Chart.js 4, JSON files for persistence.

**Design principle:** One file = one panel. Panels are flat (no hierarchy). Grouping can be added later via a `group` field without breaking the flat model.

## Data Model

### Panel Config (JSON file)

```json
{
  "id": "uuid-v4",
  "title": "Token Usage",
  "metric": "gen_ai_client_token_usage",
  "labels": {"service": "agent-gateway", "model": "claude-opus-4-8"},
  "chartType": "line",
  "step": 60
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | yes | UUID v4, generated by backend on create |
| `title` | string | yes | Display title for the chart card |
| `metric` | string | yes | Prometheus metric name (maps to `__name__`) |
| `labels` | map[string]string | no | Label filters for the PromQL query |
| `chartType` | string | yes | `"line"`, `"bar"`, or `"stat"` |
| `step` | int | no | Step interval in seconds (only meaningful for line/bar, default 60) |

### PromQL Construction

For line/bar charts (range query):
```
{metric}{labels}  →  metric_name{label1="value1",label2="value2"}
```

For stat cards (instant query at now):
```
Same promql, time defaults to server current time
```

## API Design

### New Endpoints

| Method | Path | Request Body | Response | Description |
|--------|------|-------------|----------|-------------|
| `GET` | `/api/v1/dashboards` | — | `{"panels": [...]}` | List all panel configs |
| `POST` | `/api/v1/dashboards` | Panel JSON (no id) | Panel JSON (with id) | Create a panel |
| `PUT` | `/api/v1/dashboards/{id}` | Panel JSON | Panel JSON | Update a panel |
| `DELETE` | `/api/v1/dashboards/{id}` | — | `{"status":"ok"}` | Delete a panel |
| `GET` | `/api/v1/metric-names` | — | `{"status":"success","data":["metric1",...]}` | List known metric names |

### GET /api/v1/dashboards Response

```json
{
  "panels": [
    {
      "id": "abc123",
      "title": "Token Usage",
      "metric": "gen_ai_client_token_usage",
      "labels": {"model": "claude-opus-4-8"},
      "chartType": "line",
      "step": 60
    },
    {
      "id": "def456",
      "title": "Total Requests",
      "metric": "gen_ai_client_requests_total",
      "labels": {},
      "chartType": "stat"
    }
  ]
}
```

### GET /api/v1/metric-names Response

```json
{
  "status": "success",
  "data": ["gen_ai_client_request_duration_seconds_bucket", "gen_ai_client_request_duration_seconds_count", "gen_ai_client_request_duration_seconds_sum", "gen_ai_client_requests_total", "gen_ai_client_token_usage"]
}
```

Uses the Prometheus labels response format (consistent with `/api/v1/labels`).

### Store Implementation

Panel configs stored as JSON files under `data/dashboards/`. File name: `{id}.json`.

- List: read directory, unmarshal each `.json` file
- Create: generate UUID, marshal + write new file
- Update: marshal + overwrite existing file (return 404 if not found)
- Delete: remove file (return 404 if not found)

### Metric Names Discovery

The `__name__` label is not currently in `labelIdx`. Need to add it during `Insert`: track metric names in the in-memory index alongside other labels. The `/api/v1/metric-names` handler queries `LabelValues("__name__")`.

## Frontend Design

### Route

| Path | Name | Component |
|------|------|-----------|
| `/dashboards` | `dashboards` | `Dashboard.vue` |

Sidebar nav: new "Metrics" link below "Trace".

### Dashboard.vue

**Layout:**
```
┌──────────────────────────────────────────────┐
│  [1h] [6h] [24h] [Custom...]   [+ New Panel] │  ← toolbar
├──────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐          │
│  │ Panel 1      │  │ Panel 2      │          │  ← CSS Grid
│  │ (line chart) │  │ (stat card)  │          │     2 columns
│  └──────────────┘  └──────────────┘          │
│  ┌──────────────┐  ┌──────────────┐          │
│  │ Panel 3      │  │  Empty slot   │          │
│  └──────────────┘  │              │          │
│                    └──────────────┘          │
└──────────────────────────────────────────────┘
```

**State:**
- `panels: PanelConfig[]` — loaded from API
- `timeRange: { start, end }` — computed from selected preset
- `timePreset: string` — "1h" | "6h" | "24h" | "custom"

**Data flow:**
1. `onMounted`: `GET /api/v1/dashboards` → populate `panels`
2. Each `PanelChart` receives its config + global `timeRange`
3. Time range change → all `PanelChart` re-fetch via watcher

### PanelChart.vue

**Props:** `panel: PanelConfig`, `timeRange: { start, end }`

**Behavior by chartType:**

| chartType | API | Chart.js config | X axis | Y axis |
|-----------|-----|-----------------|--------|--------|
| `line` | `query_range` | `type: 'line'` | time (HH:mm) | value |
| `bar` | `query_range` | `type: 'bar'` | time (HH:mm) | value |
| `stat` | `query` (time=now) | N/A (big text) | — | latest value |

**Card UI:**
- Title bar with panel title, edit (pencil) icon, delete (trash) icon
- Chart area (fills remaining card height)
- Loading state: spinner overlay
- Error state: red error text
- Empty state: "No data" text

### PanelForm.vue (Modal)

**Trigger:** "+ New Panel" button or edit icon on card.

**Fields:**
1. **Title** — text input, required
2. **Metric** — `<select>` dropdown, populated from `GET /api/v1/metric-names`, required
3. **Labels** — dynamic key-value rows. Each row: key dropdown (from `/api/v1/labels`), value dropdown (from `/api/v1/label/:key/values`), remove button. "+ Add label" button at bottom.
4. **Chart Type** — `<select>`: Line / Bar / Stat, required
5. **Step** — number input (seconds), only shown when chartType is line or bar, default 60

**Submit:**
- Create: `POST /api/v1/dashboards` with panel JSON
- Update: `PUT /api/v1/dashboards/{id}` with panel JSON
- On success: close modal, refresh panels list

### Time Range Selector

Preset buttons: **1h** / **6h** / **24h** / **Custom**.

Custom mode shows two datetime-local inputs (start, end). Step is auto-set to `timeRange / 60` (roughly 60 data points).

### API Client Additions

New TypeScript interfaces and functions in `web/src/api/client.ts`:

```typescript
interface PanelConfig {
  id: string;
  title: string;
  metric: string;
  labels: Record<string, string>;
  chartType: 'line' | 'bar' | 'stat';
  step?: number;
}

function listDashboards(): Promise<{ panels: PanelConfig[] }>
function createDashboard(panel: Omit<PanelConfig, 'id'>): Promise<PanelConfig>
function updateDashboard(id: string, panel: PanelConfig): Promise<PanelConfig>
function deleteDashboard(id: string): Promise<void>
function getMetricNames(): Promise<{ status: string; data: string[] }>
```

## Error Handling

- **File I/O errors (backend):** Return HTTP 500 with error message
- **Panel not found:** Return HTTP 404
- **Invalid panel JSON:** Return HTTP 400 with validation error
- **API query fails (frontend):** Show error text in the chart card area
- **Empty result from query:** Show "No data" placeholder in chart
- **Label index empty:** Return empty arrays (not errors)
